# 设计模式100题（含详细答案）

## 一、基础概念（1-15题）

### 1. 什么是设计模式？设计模式由谁首次系统化提出？
**答案**：设计模式是在软件设计中针对常见问题总结出的可复用解决方案。Erich Gamma、Richard Helm、Ralph Johnson、John Vlissides（GoF四人组）在1994年出版的《Design Patterns: Elements of Reusable Object-Oriented Software》中首次系统化提出23种设计模式。

### 2. 设计模式通常分为哪三大类？每类包含哪些模式？
**答案**：
- **创建型**（5种）：单例、工厂方法、抽象工厂、建造者、原型
- **结构型**（7种）：适配器、桥接、组合、装饰器、外观、享元、代理
- **行为型**（11种）：责任链、命令、解释器、迭代器、中介者、备忘录、观察者、状态、策略、模板方法、访问者

### 3. 设计模式的六大原则是什么？请简述每个原则的核心思想。
**答案**：
1. **单一职责原则（SRP）**：一个类只负责一个职责
2. **开闭原则（OCP）**：对扩展开放，对修改关闭
3. **里氏替换原则（LSP）**：子类可以替换父类而不影响程序正确性
4. **依赖倒置原则（DIP）**：依赖抽象而非具体实现
5. **接口隔离原则（ISP）**：接口应该小而专，而非大而全
6. **迪米特法则（LoD）**：最少知识原则，降低耦合

### 4. 什么是SOLID原则？它包含哪五个原则？
**答案**：SOLID是面向对象设计的五个基本原则缩写：
- **S**ingle Responsibility（单一职责）
- **O**pen/Closed（开闭）
- **L**iskov Substitution（里氏替换）
- **I**nterface Segregation（接口隔离）
- **D**ependency Inversion（依赖倒置）

### 5. 开闭原则（OCP）的核心思想是什么？如何在实际开发中遵循？
**答案**：对扩展开放，对修改关闭。通过抽象和多态实现，新增功能时通过添加新类而非修改现有代码。例如使用策略模式替换if-else，新增策略只需添加新类。

### 6. 里氏替换原则（LSP）要求子类必须能够替换父类，违反该原则的典型场景是什么？
**答案**：子类重写父类方法时改变了原有行为。典型场景：正方形继承长方形，修改了setWidth/setHeight的逻辑导致面积计算错误。

### 7. 依赖倒置原则（DIP）中"依赖抽象而非具体实现"的含义是什么？
**答案**：高层模块不应该依赖低层模块，两者都应该依赖抽象；抽象不应该依赖细节，细节应该依赖抽象。例如Service层依赖DAO接口而非具体实现类。

### 8. 接口隔离原则（ISP）与单一职责原则（SRP）的区别是什么？
**答案**：
- SRP关注**类的职责**单一，一个类只做一件事
- ISP关注**接口的粒度**合理，避免"胖接口"导致实现类被迫实现不需要的方法

### 9. 迪米特法则（LoD）为什么被称为"最少知识原则"？
**答案**：一个对象应当对其他对象有尽可能少的了解，只与直接的朋友通信（成员变量、方法参数、方法返回值中的对象）。避免链式调用如`a.getB().getC().doSomething()`。

### 10. 合成复用原则（CRP）与继承复用的区别和优劣？
**答案**：优先使用组合/聚合而非继承。
- **组合**：运行时动态绑定，耦合度低，更灵活
- **继承**：编译时静态绑定，耦合度高，破坏封装

### 11. 设计模式与架构模式的区别是什么？
**答案**：
- **设计模式**：解决代码级别的设计问题（类/对象关系），如单例、观察者
- **架构模式**：解决系统级别的结构问题，如MVC、微服务、分层架构

### 12. 什么是反模式（Anti-Pattern）？请举例说明。
**答案**：常见但无效或有害的解决方案：
- **上帝类（God Class）**：一个类承担过多职责
- **面条代码（Spaghetti Code）**：控制流混乱
- **金锤子（Golden Hammer）**：过度使用某一技术解决所有问题
- **拷贝粘贴编程（Copy-Paste Programming）**

### 13. 设计模式在Kotlin中与传统Java实现有何主要差异？
**答案**：
- 单例：`object`关键字一行实现
- 委托属性：`by lazy`、`by Delegates.observable`简化代理模式
- 密封类：优化策略模式和访问者模式
- 高阶函数：简化观察者模式和策略模式
- 类委托：`by`关键字实现装饰器模式

### 14. 为什么说"设计模式是经验的总结，而非发明的产物"？
**答案**：模式源于实际项目中的最佳实践，是对成功解决方案的抽象和命名，而非凭空创造。GoF的模式来自实际C++和Smalltalk项目经验。

### 15. 过度使用设计模式会导致什么问题？
**答案**：过度设计（Over-engineering），增加不必要的复杂度，降低代码可读性和维护性。应遵循KISS原则（Keep It Simple, Stupid），简单场景不需要模式。

## 二、创建型模式（16-35题）

### 16. 单例模式（Singleton）的意图是什么？请写出Kotlin的线程安全单例实现。
**答案**：确保一个类只有一个实例，并提供全局访问点。
```kotlin
// Kotlin最简实现（编译器自动生成线程安全实现）
object Singleton {
    fun doSomething() {}
}

// Java双重检查锁定
class Singleton {
    @Volatile
    private static volatile Singleton instance;
    private Singleton() {}
    public static Singleton getInstance() {
        if (instance == null) {
            synchronized (Singleton.class) {
                if (instance == null) {
                    instance = new Singleton();
                }
            }
        }
        return instance;
    }
}
```

### 17. 单例模式有哪些实现方式？各自优缺点？
**答案**：
| 方式 | 优点 | 缺点 |
|------|------|------|
| 饿汉式 | 线程安全，实现简单 | 类加载即创建，可能浪费资源 |
| 懒汉式 | 延迟加载 | 需同步，性能差 |
| 双重检查锁定（DCL） | 延迟加载，线程安全，高性能 | 实现稍复杂 |
| 静态内部类 | 延迟加载，线程安全，优雅 | 无法传参 |
| 枚举 | 防反射/序列化，最简洁 | 无法懒加载 |

### 18. 为什么枚举是实现单例的最佳方式？
**答案**：
1. JVM保证枚举实例唯一
2. 天然防反射攻击（反射无法创建枚举实例）
3. 天然防序列化攻击（序列化特殊处理）
4. 实现最简洁

### 19. 单例模式可能带来哪些问题？
**答案**：
- 全局状态难以单元测试
- 隐藏依赖关系，违反显式依赖原则
- 与依赖注入容器冲突
- 多ClassLoader环境下可能非单例
- 分布式系统中无法保证单例

### 20. 工厂方法模式（Factory Method）与简单工厂模式的区别？
**答案**：
- **简单工厂**：一个工厂类根据参数创建不同产品，违反开闭原则（新增产品需修改工厂）
- **工厂方法**：将创建逻辑延迟到子类，每个具体工厂对应一种产品，符合开闭原则

### 21. 抽象工厂模式（Abstract Factory）与工厂方法模式的区别？
**答案**：
- **工厂方法**：创建**单一产品**，关注"一个产品对象的创建"
- **抽象工厂**：创建**产品族**（一系列相关的产品对象），如UI组件库中的Button+TextField+Checkbox

### 22. 建造者模式（Builder）适用于什么场景？
**答案**：
- 创建复杂对象，对象有很多可选参数
- 构造过程需要分步骤进行
- 需要创建不可变对象
- 典型例子：Android AlertDialog.Builder、OkHttp.Request.Builder

### 23. Kotlin中如何使用`apply`函数简化建造者模式？
**答案**：
```kotlin
val person = Person().apply {
    name = "Tom"
    age = 20
    address = "Beijing"
}
// 或使用DSL风格的构建器
val html = buildString {
    append("<html>")
    append("<body>Hello</body>")
    append("</html>")
}
```

### 24. 原型模式（Prototype）的核心思想是什么？
**答案**：通过复制现有对象来创建新对象，而非通过new。适用于：
- 创建成本高（如需要数据库查询）
- 需要保留对象当前状态
- 类在运行时指定

### 25. 深拷贝与浅拷贝的区别？原型模式中如何实现深拷贝？
**答案**：
- **浅拷贝**：只复制对象本身，引用类型成员共享引用
- **深拷贝**：复制对象及所有引用对象

实现方式：
1. 序列化（Serializable）
2. 递归clone()
3. 拷贝构造器
4. JSON转换（如Gson）

### 26. 在Android开发中，Intent的克隆使用了哪种设计模式？
**答案**：原型模式。`Intent`实现了`Cloneable`接口，`Intent.clone()`返回副本。

### 27. 工厂方法模式在Spring框架中的典型应用是什么？
**答案**：`BeanFactory`和`ApplicationContext`。通过配置文件或注解决定创建哪个Bean，如`@Bean`方法、FactoryBean接口。

### 28. 抽象工厂模式在UI框架中的典型应用是什么？
**答案**：跨平台UI库，如Java的AWT/Swing中，不同操作系统提供不同的组件工厂（WindowsFactory、MacFactory），创建对应风格的Button、Menu等。

### 29. 建造者模式与工厂模式的区别？
**答案**：
- **工厂模式**：关注"创建什么"，一步到位创建对象
- **建造者模式**：关注"如何创建"，分步骤构建，可配置

### 30. Kotlin的`data class`的`copy()`方法体现了哪种设计模式？
**答案**：原型模式。基于现有对象创建副本，并可修改部分属性：
```kotlin
data class User(val name: String, val age: Int)
val user2 = user1.copy(age = 25)
```

### 31. 单例模式与依赖注入容器（如Dagger/Hilt）的关系？
**答案**：DI容器管理单例生命周期，通常使用`@Singleton`注解标记，由容器保证唯一性。比手动单例更灵活，支持作用域控制和测试替换。

### 32. 为什么`Calendar.getInstance()`是工厂方法模式而非单例模式？
**答案**：每次调用可能返回不同实例（根据时区/地区参数），且创建逻辑封装在方法中，由子类决定具体创建哪个Calendar实现。

### 33. 创建型模式的核心关注点是什么？
**答案**：对象的创建过程与使用分离，降低耦合，提高灵活性。让程序在创建对象时更灵活，不需要直接依赖具体类。

### 34. 在Kotlin中，如何利用`sealed class`优化工厂模式？
**答案**：
```kotlin
sealed class Animal {
    abstract fun speak()
}
class Dog : Animal() { override fun speak() = println("Woof") }
class Cat : Animal() { override fun speak() = println("Meow") }

// 类型安全的工厂，when无需else分支
fun createAnimal(type: String): Animal = when(type) {
    "dog" -> Dog()
    "cat" -> Cat()
    else -> throw IllegalArgumentException()
}
```

### 35. 对象池模式（Object Pool）属于GoF的23种模式吗？它解决了什么问题？
**答案**：不属于GoF23种。解决频繁创建销毁对象带来的性能问题，复用对象而非重复创建。典型应用：数据库连接池、线程池、Bitmap对象池。

## 三、结构型模式（36-55题）

### 36. 适配器模式（Adapter）的意图是什么？类适配器与对象适配器的区别？
**答案**：将不兼容接口转换为可兼容接口。
- **类适配器**：通过继承实现（Kotlin/Java单继承限制）
- **对象适配器**：通过组合实现（更灵活，推荐）

### 37. 装饰器模式（Decorator）与继承的区别？
**答案**：
- **装饰器**：动态添加功能，运行时组合，更灵活
- **继承**：静态添加功能，编译时确定，耦合度高

### 38. Java IO流中使用了哪些设计模式？
**答案**：
- **装饰器模式**：`BufferedInputStream`包装`FileInputStream`，`DataInputStream`再包装
- **适配器模式**：`InputStreamReader`将字节流适配为字符流

### 39. 代理模式（Proxy）有哪些类型？各自用途？
**答案**：
- **静态代理**：编译时确定，代码冗余
- **动态代理**：运行时生成，JDK动态代理（基于接口）、CGLIB（基于继承）
- **远程代理**：访问远程对象（RMI）
- **虚拟代理**：延迟加载大对象
- **保护代理**：权限控制
- **智能引用**：引用计数、日志记录

### 40. JDK动态代理与CGLIB代理的区别？
**答案**：
| 特性 | JDK动态代理 | CGLIB |
|------|------------|-------|
| 基础 | 接口 | 继承 |
| 实现 | 反射 | 字节码生成 |
| 性能 | 较慢 | 较快 |
| Spring默认 | 有接口时用 | 无接口时用 |

### 41. 外观模式（Facade）与中介者模式（Mediator）的区别？
**答案**：
- **外观模式**：简化子系统接口，单向封装，客户端通过外观访问子系统
- **中介者模式**：解耦对象间交互，多向协调，对象间不直接通信

### 42. 桥接模式（Bridge）的核心思想是什么？
**答案**：将抽象与实现分离，使两者可以独立变化。如JDBC中，`DriverManager`（抽象）与具体数据库驱动（实现）分离，切换数据库只需更换驱动。

### 43. 组合模式（Composite）适用于什么场景？
**答案**：树形结构，统一处理单个对象和组合对象。典型应用：
- 文件系统（文件和文件夹）
- UI组件树（View和ViewGroup）
- 组织架构

### 44. 享元模式（Flyweight）如何优化内存使用？
**答案**：共享细粒度对象，将对象状态分为：
- **内部状态（Intrinsic）**：共享，不随环境变化
- **外部状态（Extrinsic）**：客户端传入，随环境变化

典型应用：Java字符串常量池、Integer缓存（-128~127）、围棋棋子对象池。

### 45. Kotlin的`by lazy`委托体现了哪种设计模式？
**答案**：代理模式（虚拟代理），延迟初始化直到首次访问。
```kotlin
val heavyObject by lazy { HeavyObject() } // 首次访问时才创建
```

### 46. Android中的`ContextWrapper`使用了哪种设计模式？
**答案**：装饰器模式。包装Context并可以添加额外功能，如`Activity`继承自`ContextThemeWrapper`。

### 47. Retrofit中使用了哪些结构型模式？
**答案**：
- **代理模式**：动态代理生成API接口的实现
- **适配器模式**：`CallAdapter`适配不同返回类型（Call、Observable、Flow等）

### 48. 外观模式在Android开发中的典型应用？
**答案**：`MediaPlayer`类封装了音频播放的复杂子系统调用，提供简洁的start()/pause()/stop()接口。

### 49. 享元模式与单例模式的区别？
**答案**：
- **单例**：整个系统只有一个实例
- **享元**：可以有多个实例，但共享相同内部状态，按状态分类管理

### 50. Kotlin的`by`关键字在委托属性中体现了哪种模式？
**答案**：代理模式（Delegation）。将属性的get/set委托给另一个对象：
```kotlin
var name: String by Delegates.observable("") { prop, old, new ->
    println("$old -> $new")
}
```

### 51. 装饰器模式在Java集合框架中的体现？
**答案**：
```java
List<String> syncList = Collections.synchronizedList(new ArrayList<>());
List<String> unmodList = Collections.unmodifiableList(new ArrayList<>());
```

### 52. 桥接模式与策略模式的区别？
**答案**：
- **桥接模式**：关注抽象与实现的分离（结构层面），如形状与颜色的分离
- **策略模式**：关注算法的替换（行为层面），如不同排序算法

### 53. 为什么Kotlin的类委托（Class Delegation）是装饰器模式的语法糖？
**答案**：
```kotlin
class Derived(b: Base) : Base by b {
    // 自动转发所有方法给b
    override fun print() { // 只需重写需要装饰的方法
        println("Before")
        b.print()
        println("After")
    }
}
```

### 54. Android中的`ViewGroup`与`View`体现了哪种结构型模式？
**答案**：组合模式。`ViewGroup`是组合节点（可包含子View），`View`是叶子节点，统一继承自`View`。

### 55. 在微服务架构中，API Gateway使用了哪种设计模式？
**答案**：外观模式。为客户端提供统一入口，隐藏后端服务的复杂性，处理路由、认证、限流等。

## 四、行为型模式（56-85题）

### 56. 观察者模式（Observer）的意图是什么？
**答案**：定义对象间的一对多依赖，当一个对象（Subject）状态改变时，所有依赖者（Observer）自动收到通知并更新。

### 57. Kotlin中`Flow`与LiveData如何体现观察者模式？
**答案**：
- **Flow**：冷流，订阅时才开始发射数据，每个订阅者独立接收
- **LiveData**：生命周期感知的观察者模式，自动管理订阅生命周期

### 58. 发布-订阅模式与观察者模式的区别？
**答案**：
| 特性 | 观察者模式 | 发布-订阅模式 |
|------|-----------|-------------|
| 耦合度 | 直接耦合（Subject知道Observer） | 完全解耦（通过事件中心） |
| 通信方式 | 同步 | 异步 |
| 中间层 | 无 | 事件总线/消息队列 |
| 典型实现 | Java Observable | EventBus、RxBus、MQTT |

### 59. 策略模式（Strategy）的核心思想？
**答案**：定义算法族，分别封装，让它们可以互相替换。策略让算法的变化独立于使用算法的客户。
典型应用：`Comparator`接口、支付方式选择、压缩算法选择。

### 60. Kotlin中如何使用高阶函数简化策略模式？
**答案**：
```kotlin
// 无需定义策略接口，直接用函数类型
val strategies = mapOf(
    "add" to { a: Int, b: Int -> a + b },
    "subtract" to { a: Int, b: Int -> a - b }
)
val result = strategies["add"]?.invoke(3, 5) // 8
```

### 61. 模板方法模式（Template Method）与策略模式的区别？
**答案**：
- **模板方法**：通过继承固定算法骨架，子类重写部分步骤（is-a关系）
- **策略模式**：通过组合替换整个算法（has-a关系）

### 62. Android中的`AsyncTask`使用了哪种设计模式？
**答案**：模板方法模式。`doInBackground`、`onProgressUpdate`、`onPostExecute`是模板方法，子类按固定流程执行。

### 63. 命令模式（Command）的意图是什么？
**答案**：将请求封装为对象，支持：
- 参数化客户端
- 请求队列
- 操作日志
- 撤销（Undo）操作

### 64. Android中的`Handler`与`Message`体现了哪种模式？
**答案**：命令模式。`Message`封装命令和参数，`Handler`作为接收者执行命令。

### 65. 职责链模式（Chain of Responsibility）适用于什么场景？
**答案**：多个对象有机会处理请求，避免发送者与接收者耦合。典型应用：
- Android事件分发机制
- OkHttp拦截器链
- 审批流程

### 66. OkHttp的拦截器链使用了哪种设计模式？
**答案**：职责链模式（Chain of Responsibility）。请求按顺序经过多个拦截器处理，每个拦截器决定是否处理或传递给下一个。

### 67. 状态模式（State）与策略模式的区别？
**答案**：
- **状态模式**：状态内部可切换，状态决定行为，状态间有转换关系
- **策略模式**：策略由客户端选择，策略间独立，无转换关系

### 68. 迭代器模式（Iterator）在Kotlin中的体现？
**答案**：
- `Iterator`接口：`hasNext()`、`next()`
- `for-in`循环语法糖
- `Sequence`惰性迭代

### 69. 访问者模式（Visitor）解决了什么问题？
**答案**：在不修改类的前提下，为类层次结构添加新操作。适用于：
- 编译器的AST遍历
- 报表生成（遍历不同节点生成不同格式）
- 操作与数据结构分离

### 70. Kotlin中`sealed class`与访问者模式的关系？
**答案**：`when`表达式配合`sealed class`实现类型安全的访问者模式，编译器确保所有子类都被处理：
```kotlin
sealed class Expr
data class Num(val value: Int) : Expr()
data class Add(val left: Expr, val right: Expr) : Expr()

fun eval(e: Expr): Int = when(e) { // 无需else，编译器检查完备性
    is Num -> e.value
    is Add -> eval(e.left) + eval(e.right)
}
```

### 71. 备忘录模式（Memento）如何实现状态保存与恢复？
**答案**：捕获对象内部状态并外部化存储，支持撤销。三个角色：
- **Originator**：需要保存状态的对象
- **Memento**：状态存储对象
- **Caretaker**：负责保存和恢复

### 72. Android中`onSaveInstanceState`/`onRestoreInstanceState`体现了哪种模式？
**答案**：备忘录模式。系统保存Activity状态到Bundle，恢复时从中读取。

### 73. 中介者模式（Mediator）在Android中的典型应用？
**答案**：
- `ViewModel`作为Fragment间通信的中介者
- `CoordinatorLayout.Behavior`协调子View交互
- Room数据库的`MediatorLiveData`

### 74. 解释器模式（Interpreter）适用于什么场景？
**答案**：定义语言的文法表示，并解释句子。适用于：
- 正则表达式引擎
- 简单的DSL解析
- 数学表达式计算
- SQL解析

### 75. Kotlin DSL的实现依赖于哪些设计模式？
**答案**：
- **构建者模式**：`@DslMarker`注解控制作用域
- **Lambda with receiver**：`T.() -> Unit`提供上下文
- **扩展函数**：丰富DSL表达能力

### 76. 空对象模式（Null Object Pattern）属于GoF23种吗？它解决了什么问题？
**答案**：不属于GoF23种。解决空指针异常问题，提供默认无行为实现：
```kotlin
// 而非返回null，返回空实现
fun getLogger(): Logger = if (debug) FileLogger() else NullLogger()
```

### 77. Android中的`RecyclerView.Adapter`使用了哪种设计模式？
**答案**：适配器模式。将数据集合适配到RecyclerView的显示需求，统一接口`onCreateViewHolder`、`onBindViewHolder`。

### 78. Kotlin的`Sequence`与`Iterable`的区别体现了哪种设计思想？
**答案**：迭代器模式的优化。`Sequence`惰性求值，避免中间集合创建，适合大数据量处理：
```kotlin
list.asSequence()
    .filter { it > 0 }  // 不创建新集合
    .map { it * 2 }     // 不创建新集合
    .toList()           // 终端操作时才计算
```

### 79. 责任链模式在Android触摸事件分发中的应用？
**答案**：`Activity -> Window -> DecorView -> ViewGroup -> View`，事件沿责任链传递：
- `onInterceptTouchEvent`：是否拦截
- `onTouchEvent`：是否消费

### 80. 策略模式在Android动画中的体现？
**答案**：`Interpolator`接口，不同插值策略：
- `LinearInterpolator`：匀速
- `AccelerateInterpolator`：加速
- `DecelerateInterpolator`：减速
- `BounceInterpolator`：弹跳

### 81. Kotlin的`sealed interface`如何优化状态模式？
**答案**：定义有限状态集合，配合`when`表达式exhaustive check：
```kotlin
sealed interface State {
    data object Idle : State
    data class Loading(val progress: Int) : State
    data class Success(val data: String) : State
    data class Error(val message: String) : State
}
```

### 82. 命令模式如何支持撤销（Undo）功能？
**答案**：
1. 维护命令历史栈
2. 每个命令实现`execute()`和`undo()`
3. 撤销时弹出栈顶命令，调用`undo()`

### 83. 观察者模式中推模型与拉模型的区别？
**答案**：
- **推模型（Push）**：Subject发送完整数据给Observer，如`onNext(data)`
- **拉模型（Pull）**：Subject发送通知，Observer主动拉取数据，如`notifyObservers()`后调用`getData()`

### 84. 模板方法模式中的"钩子方法（Hook Method）"是什么？
**答案**：父类中定义的空方法或默认实现，子类可选择性重写以扩展行为，不影响算法骨架：
```kotlin
abstract class Game {
    fun play() { // 模板方法
        initialize()
        startPlay()
        endPlay()
        hook() // 钩子方法，可选重写
    }
    open fun hook() {} // 默认空实现
}
```

### 85. Android Jetpack的`Lifecycle`组件如何体现观察者模式？
**答案**：`LifecycleObserver`观察`LifecycleOwner`状态变化（ON_CREATE、ON_START等），自动管理生命周期，避免内存泄漏。

## 五、综合应用与高级话题（86-100题）

### 86. MVC、MVP、MVVM三种架构模式的核心区别？
**答案**：
| 特性 | MVC | MVP | MVVM |
|------|-----|-----|------|
| View与Model | 直接通信 | 不直接通信 | 不直接通信 |
| Presenter/VM | Controller处理逻辑 | Presenter持有View接口 | ViewModel不持有View |
| 数据绑定 | 无 | 无 | 双向数据绑定 |
| 测试性 | 差 | 好 | 好 |
| Android代表 | 早期Activity | 手动接口回调 | Jetpack + DataBinding |

### 87. MVVM在Android中如何通过Jetpack实现？涉及哪些设计模式？
**答案**：
- **ViewModel**：工厂模式创建（`ViewModelProvider.Factory`）
- **LiveData**：观察者模式
- **DataBinding**：观察者模式 + 适配器模式
- **Repository**：抽象工厂/策略模式

### 88. MVI（Model-View-Intent）与MVVM的区别？
**答案**：
- MVI强调**单向数据流**和**不可变状态**
- 通过Intent驱动状态变化
- 状态集中管理（单一State对象）
- 更容易追踪和调试状态变化

### 89. 依赖注入（DI）与控制反转（IoC）的关系？
**答案**：DI是实现IoC的一种方式。IoC是设计原则（控制权从应用代码转移到框架），DI是具体实现（通过构造函数、Setter、接口注入依赖）。

### 90. Kotlin的`Koin`与`Dagger/Hilt`在实现DI时分别使用了哪些设计模式？
**答案**：
- **Koin**：DSL（构建者模式 + 扩展函数），纯Kotlin实现，运行时解析
- **Dagger/Hilt**：编译时代码生成（抽象工厂 + 代理模式），编译期检查

### 91. RxJava中使用了哪些设计模式？
**答案**：
- **观察者模式**：Observable/Observer
- **迭代器模式**：Flowable支持背压
- **装饰器模式**：操作符链式调用（map、filter等包装Observable）
- **策略模式**：不同Scheduler策略

### 92. Kotlin协程的`Channel`与`Flow`分别体现了哪些并发设计模式？
**答案**：
- **Channel**：生产者-消费者模式（有缓冲或无缓冲）
- **Flow**：响应式流（观察者模式 + 迭代器模式），冷流，支持背压

### 93. 什么是反应式编程（Reactive Programming）？它基于哪些设计模式？
**答案**：基于数据流和变化传播的编程范式，核心设计模式：
- 观察者模式（数据变化通知）
- 迭代器模式（流式处理）
- 函数式编程（纯函数、不可变数据）

### 94. 设计模式在Kotlin中如何与函数式编程结合？
**答案**：
- **策略模式** → 高阶函数
- **装饰器模式** → 函数组合（compose/andThen）
- **访问者模式** → `when`表达式 + sealed class
- **命令模式** → lambda表达式
- **观察者模式** → Flow/Sequence

### 95. Android中的`WorkManager`使用了哪些设计模式？
**答案**：
- **工厂模式**：`WorkRequest.Builder`创建任务
- **命令模式**：`Worker`封装任务逻辑
- **观察者模式**：`LiveData<WorkInfo>`监听任务状态
- **状态模式**：WorkInfo的状态转换（ENQUEUED -> RUNNING -> SUCCEEDED）

### 96. Room数据库中使用了哪些设计模式？
**答案**：
- **抽象工厂**：`Room.databaseBuilder()`创建数据库
- **代理模式**：DAO接口由编译器生成实现类
- **适配器模式**：`TypeConverter`转换自定义类型
- **观察者模式**：返回`LiveData`/`Flow`支持数据变化监听

### 97. 如何评价"设计模式是面向对象语言的补丁"这一说法？
**答案**：有一定道理。函数式语言（如Haskell、Kotlin）中许多模式天然由语言特性支持：
- 单例 → `object`
- 策略 → 高阶函数
- 访问者 → 模式匹配
- 装饰器 → 函数组合
- 空对象 → Maybe/Option类型

### 98. 在Kotlin多平台（KMP）开发中，设计模式的选择有何特殊考虑？
**答案**：
- 需考虑平台差异，常用**抽象工厂**/**策略模式**封装平台特定实现
- `expect`/`actual`机制体现了**桥接模式**思想
- 共享逻辑层使用**外观模式**封装平台API差异

### 99. 如何根据场景选择合适的设计模式？决策流程是什么？
**答案**：
1. **明确问题类型**：创建/结构/行为
2. **分析变化点**：什么在变化？（算法？结构？状态？）
3. **匹配模式意图**：阅读模式意图描述
4. **评估复杂度收益比**：简单场景不用模式
5. **考虑语言特性**：Kotlin可用高阶函数/委托等简化

### 100. 设计模式在现代软件开发中的发展趋势？
**答案**：
1. **向函数式/反应式演进**：FP语言特性替代传统模式
2. **与框架深度集成**：Spring/Jetpack内置模式实现
3. **DSL化**：Kotlin DSL、Gradle DSL等
4. **模式语言扩展**：并发模式、分布式模式、云原生模式
5. **反模式识别自动化**：静态分析工具检测
6. **AI辅助设计**：大模型推荐合适的模式组合

---

> **总结**：设计模式是前人智慧的结晶，但不应教条化。在Kotlin等现代语言中，很多模式已被语言特性优雅替代。理解模式背后的设计思想（解耦、复用、扩展）比记住具体实现更重要。
