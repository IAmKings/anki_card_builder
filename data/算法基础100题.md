## 一、数据结构基础（1-20题）

### 1. 数组和链表有什么区别？各自的优缺点是什么？
**答：**
| 特性 | 数组 | 链表 |
|------|------|------|
| 存储方式 | 连续内存 | 离散内存（节点+指针） |
| 随机访问 | O(1) | O(n) |
| 插入/删除 | O(n)（需移动元素） | O(1)（已知位置） |
| 内存利用 | 固定大小，可能浪费 | 动态分配，额外指针开销 |
| 缓存友好 | 好（空间局部性） | 差（指针跳转） |
| 适用场景 | 频繁访问、大小固定 | 频繁插入删除、大小不定 |

### 2. 栈（Stack）和队列（Queue）有什么区别？
**答：**
- **栈**：LIFO（后进先出），操作在栈顶进行。应用：函数调用栈、表达式求值、括号匹配、浏览器前进后退
- **队列**：FIFO（先进先出），操作在队头和队尾进行。应用：BFS、任务调度、缓冲区、打印队列
- **双端队列（Deque）**：两端都可插入删除，兼具栈和队列特性

### 3. 什么是哈希表（Hash Table）？如何解决哈希冲突？
**答：** 哈希表通过哈希函数将键映射到数组索引，实现O(1)平均时间复杂度的查找。
**冲突解决方法：**
- **开放寻址法**：线性探测、二次探测、双重哈希
- **链地址法（拉链法）**：每个桶维护一个链表，冲突元素链入链表
- **再哈希法**：使用多个哈希函数
- **建立公共溢出区**

### 4. 哈希表的装载因子（Load Factor）是什么？有什么影响？
**答：** 装载因子 = 元素个数 / 桶的数量。影响：
- 装载因子越大，冲突概率越高，性能下降
- 装载因子过小，空间浪费
- 通常阈值设为0.75，超过时进行扩容（rehash），时间复杂度O(n)

### 5. 二叉树（Binary Tree）有哪些基本类型？
**答：**
- **满二叉树**：每层节点数达到最大，所有叶子节点在同一层
- **完全二叉树**：除最后一层外完全填满，最后一层节点集中在左侧
- **二叉搜索树（BST）**：左子树所有节点 < 根节点 < 右子树所有节点
- **平衡二叉树（AVL）**：任意节点左右子树高度差不超过1
- **红黑树**：近似平衡的二叉搜索树，插入删除效率更高

### 6. 二叉搜索树（BST）的查找、插入、删除时间复杂度是多少？
**答：**
- **查找**：平均O(log n)，最坏O(n)（退化为链表）
- **插入**：平均O(log n)，最坏O(n)
- **删除**：平均O(log n)，最坏O(n)
- 平衡BST（AVL/红黑树）保证O(log n)

### 7. AVL树和红黑树有什么区别？
**答：**
| 特性 | AVL树 | 红黑树 |
|------|-------|--------|
| 平衡度 | 严格平衡（高度差≤1） | 近似平衡（黑高相同） |
| 查找效率 | 更快（树更矮） | 稍慢 |
| 插入/删除 | 旋转次数多，维护成本高 | 旋转次数少，最多2次 |
| 适用场景 | 查找频繁、插入删除少 | 插入删除频繁（如C++ map/set） |

### 8. B树和B+树有什么区别？为什么数据库索引常用B+树？
**答：**
- **B树**：所有节点都存储数据，叶子节点不相连
- **B+树**：只有叶子节点存储数据，内部节点只存键值，叶子节点通过指针相连
**B+树优势：**
- 内部节点更小，一个磁盘页能存更多索引，树更矮，IO次数更少
- 叶子节点相连，范围查询和遍历只需顺序访问叶子节点
- 查询性能稳定（总是到叶子节点）

### 9. 堆（Heap）是什么？最大堆和最小堆的区别？
**答：** 堆是完全二叉树，满足堆性质：
- **最大堆**：父节点 ≥ 子节点，堆顶是最大值
- **最小堆**：父节点 ≤ 子节点，堆顶是最小值
**应用**：优先队列、Top K问题、堆排序、Dijkstra算法

### 10. 堆的插入和删除操作时间复杂度是多少？
**答：**
- **插入**：O(log n)，插入到末尾后向上调整（sift up）
- **删除堆顶**：O(log n)，用末尾元素替换堆顶后向下调整（sift down）
- **建堆**：O(n)（非O(n log n)，通过从最后一个非叶子节点向下调整）

### 11. 优先队列（Priority Queue）通常用什么数据结构实现？
**答：** 通常用**堆（Heap）**实现：
- 插入：O(log n)
- 取出最高优先级元素：O(log n)
- 查看最高优先级：O(1)
- 也可用有序数组（插入O(n)，取出O(1)）或二叉搜索树

### 12. 图（Graph）的存储方式有哪些？各自的优缺点？
**答：**
| 存储方式 | 空间复杂度 | 适用场景 |
|----------|-----------|----------|
| **邻接矩阵** | O(V²) | 稠密图、频繁查询边是否存在 |
| **邻接表** | O(V+E) | 稀疏图、遍历邻居 |
| **边列表** | O(E) | Kruskal算法、需要按边排序 |
| **十字链表** | O(V+E) | 有向图，同时找入边和出边 |

### 13. 有向图和无向图的区别？如何表示？
**答：**
- **无向图**：边无方向，(u,v)和(v,u)是同一条边，邻接矩阵对称
- **有向图**：边有方向，(u,v)和(v,u)是不同边，邻接矩阵不对称
- **表示**：邻接矩阵/邻接表，有向图需分别存储入边和出边（或十字链表）

### 14. 什么是拓扑排序（Topological Sort）？适用什么场景？
**答：** 对有向无环图（DAG）的节点进行线性排序，使得对每条边(u,v)，u在v之前。算法：Kahn算法（BFS，入度为0的节点入队）或DFS（后序遍历逆序）。应用：课程选修计划、任务调度、编译依赖、Makefile构建。

### 15. 并查集（Union-Find）是什么？如何实现路径压缩和按秩合并？
**答：** 用于管理元素分组的数据结构，支持：
- **Find**：查找元素所在集合的代表（根节点）
- **Union**：合并两个集合
**优化：**
- **路径压缩**：Find时把路径上所有节点直接指向根，时间复杂度近似O(α(n))（阿克曼函数反函数，<5）
- **按秩合并**：合并时把矮树合并到高树下，避免树变高

### 16. 跳表（Skip List）是什么？时间复杂度如何？
**答：** 跳表是有序链表的多层索引结构，通过随机化实现平衡。时间复杂度：
- 查找/插入/删除：平均O(log n)，最坏O(n)
- 空间复杂度：O(n)
- 优势：实现简单，支持区间查询，并发性能好（Redis有序集合使用）

### 17. 线段树（Segment Tree）是什么？能解决什么问题？
**答：** 线段树是平衡二叉树，每个节点代表一个区间，用于区间查询和修改。时间复杂度：
- 建树：O(n)
- 区间查询：O(log n)
- 单点/区间更新：O(log n)
- 应用：区间最值、区间和、区间覆盖、RMQ问题

### 18. 树状数组（Fenwick Tree / BIT）是什么？与线段树的区别？
**答：** 树状数组用于高效计算前缀和，支持单点更新和前缀查询。时间复杂度：
- 更新：O(log n)
- 前缀查询：O(log n)
- 区间查询：两次前缀查询，O(log n)
- **与线段树区别**：树状数组代码更简洁、常数更小，但只能处理满足"区间减法"的问题（如求和），不能处理区间最值等；线段树更通用

### 19. 字典树（Trie）是什么？有什么应用？
**答：** Trie是前缀树，用于高效存储和检索字符串集合。节点代表字符，路径代表字符串。时间复杂度：
- 插入/查找：O(L)，L为字符串长度
- 应用：自动补全、拼写检查、IP路由最长前缀匹配、字符串统计

### 20. 什么是LRU缓存（Least Recently Used）？如何实现？
**答：** 当缓存满时淘汰最久未使用的数据。实现方式：
- **哈希表 + 双向链表**：哈希表O(1)查找，双向链表O(1)移动节点到头部/删除尾部
- 访问时：哈希表找到节点，移到链表头部
- 插入时：如果满，删除尾部节点（最久未使用），插入头部
- 时间复杂度：get/put都是O(1)

## 二、排序算法（21-35题）

### 21. 常见排序算法的时间复杂度和空间复杂度对比？
**答：**
| 算法 | 平均时间 | 最坏时间 | 最好时间 | 空间 | 稳定性 |
|------|----------|----------|----------|------|--------|
| 冒泡排序 | O(n²) | O(n²) | O(n) | O(1) | ✓ |
| 选择排序 | O(n²) | O(n²) | O(n²) | O(1) | ✗ |
| 插入排序 | O(n²) | O(n²) | O(n) | O(1) | ✓ |
| 希尔排序 | O(n^1.3) | O(n²) | O(n) | O(1) | ✗ |
| 归并排序 | O(n log n) | O(n log n) | O(n log n) | O(n) | ✓ |
| 快速排序 | O(n log n) | O(n²) | O(n log n) | O(log n) | ✗ |
| 堆排序 | O(n log n) | O(n log n) | O(n log n) | O(1) | ✗ |
| 计数排序 | O(n+k) | O(n+k) | O(n+k) | O(k) | ✓ |
| 桶排序 | O(n+k) | O(n²) | O(n) | O(n+k) | ✓ |
| 基数排序 | O(d(n+k)) | O(d(n+k)) | O(d(n+k)) | O(n+k) | ✓ |

### 22. 快速排序（Quick Sort）的基本思想和步骤？
**答：** 分治思想：
1. **选择基准（Pivot）**：通常选最后一个元素或随机元素
2. **分区（Partition）**：将数组分为小于基准和大于基准的两部分
3. **递归排序**：对左右两部分递归进行快速排序
**优化**：随机选基准、三数取中、尾递归优化、小数组用插入排序

### 23. 快速排序的最坏情况是什么？如何避免？
**答：** 最坏情况：数组已有序或逆序，每次分区极度不平衡，时间复杂度O(n²)。避免方法：
- **随机化**：随机选择基准元素
- **三数取中**：取首、中、尾三个元素的中位数作为基准
- **三路划分**：处理大量重复元素（荷兰国旗问题）

### 24. 归并排序（Merge Sort）和快速排序的对比？
**答：**
| 特性 | 归并排序 | 快速排序 |
|------|----------|----------|
| 稳定性 | 稳定 | 不稳定 |
| 最坏时间 | O(n log n) | O(n²) |
| 空间复杂度 | O(n) | O(log n) |
| 适用场景 | 链表排序、外部排序 | 数组排序、通用场景 |
| 缓存性能 | 较差（需要额外数组） | 较好（原地分区） |

### 25. 堆排序（Heap Sort）的步骤是什么？
**答：**
1. **建堆**：将无序数组构建成最大堆，O(n)
2. **排序**：重复以下步骤：
   - 将堆顶（最大值）与末尾元素交换
   - 堆大小减1
   - 对新的堆顶进行sift down，恢复堆性质
3. 时间复杂度：O(n log n)，空间复杂度：O(1)

### 26. 计数排序（Counting Sort）和基数排序（Radix Sort）的适用场景？
**答：**
- **计数排序**：数据范围小（k << n）的整数排序，如年龄排序（0-150）
- **基数排序**：数据范围大但位数固定，如电话号码、身份证号排序
- 都是非比较排序，时间复杂度O(n)，但受数据范围限制

### 27. 什么是稳定排序？为什么稳定性重要？
**答：** 稳定排序保证相等元素的相对顺序不变。重要性：
- 多关键字排序：如先按年龄排序，再按姓名排序，稳定排序保持年龄相同者的姓名有序
- 实际应用：数据库排序、Excel多列排序
- 稳定排序：冒泡、插入、归并、计数、基数、桶排序
- 不稳定排序：选择、希尔、快速、堆排序

### 28. 如何在O(n)时间复杂度内找到数组中的第K大元素？
**答：**
- **快速选择（Quick Select）**：基于快速排序的分区思想，平均O(n)，最坏O(n²)
- **中位数的中位数**：保证O(n)最坏时间，但常数较大
- **最小堆**：维护大小为K的最小堆，O(n log K)，适合K较小或数据流场景

### 29. 荷兰国旗问题（Dutch National Flag Problem）是什么？如何解决？
**答：** 将数组按三种颜色（或值）排序，如按0、1、2排序。三路划分：
- 指针low：0的右边界
- 指针high：2的左边界
- 指针current：当前遍历位置
- current=0：与low交换，low++，current++
- current=1：current++
- current=2：与high交换，high--（current不++，因为交换过来的元素未知）

### 30. 外部排序（External Sort）是什么？如何处理海量数据排序？
**答：** 数据量太大无法全部装入内存时的排序方法：
1. **分块**：将数据分成多个能装入内存的块
2. **内部排序**：对每个块分别排序，写入临时文件
3. **多路归并**：使用最小堆进行K路归并，合并所有有序块
- 优化：置换-选择排序生成更长的初始有序段、败者树优化归并

### 31. 快速排序的分区算法（Lomuto vs Hoare）有什么区别？
**答：**
- **Lomuto分区**：单向扫描，简单但交换次数多，基准通常选末尾
- **Hoare分区**：双向扫描，交换次数少，效率更高，但基准位置不确定
- 实际实现多使用Hoare分区或Lomuto的变体

### 32. 希尔排序（Shell Sort）的增量序列如何选择？
**答：** 希尔排序是插入排序的改进，通过增量分组排序。增量序列：
- **Shell原始序列**：n/2, n/4, ..., 1（最坏O(n²)）
- **Hibbard序列**：2^k-1，最坏O(n^1.5)
- **Sedgewick序列**：更复杂，最坏O(n^1.3)
- 增量序列的选择直接影响性能

### 33. TimSort是什么？为什么Python和Java的排序使用它？
**答：** TimSort是归并排序和插入排序的混合算法：
- 先找出数组中的有序段（run），用插入排序扩展或生成短run
- 用归并排序合并run，合并时做优化（galloping mode）
- 优势：对真实数据（部分有序）性能极好，最坏O(n log n)，稳定排序
- Python（list.sort()）、Java（Arrays.sort()对象数组）、Android使用

### 34. 排序算法中如何选择合适的算法？
**答：**
- **小数据量（n < 50）**：插入排序
- **基本有序**：插入排序、TimSort
- **大数据量随机**：快速排序（优化版）
- **需要稳定性**：归并排序、TimSort
- **空间受限**：堆排序、希尔排序
- **数据范围小**：计数排序、基数排序
- **链表排序**：归并排序
- **外部排序**：多路归并

### 35. 如何证明基于比较的排序算法时间复杂度下界是O(n log n)？
**答：** 通过决策树模型：
- n个元素的排列有n!种可能
- 决策树是二叉树，每个叶子代表一种排列
- 树高h ≥ log₂(n!) ≈ n log₂n - n log₂e + O(log n)
- 由斯特林公式，log₂(n!) = Θ(n log n)
- 因此任何基于比较的排序最坏情况至少需要Ω(n log n)次比较

## 三、查找算法（36-45题）

### 36. 二分查找（Binary Search）的基本思想和时间复杂度？
**答：** 在有序数组中，每次将查找范围缩小一半：
- 比较中间元素与目标值
- 如果相等，返回；如果目标更小，在左半部分继续；否则在右半部分
- 时间复杂度：O(log n)
- 空间复杂度：O(1)（迭代版）或O(log n)（递归版）

### 37. 二分查找的边界条件容易出错，如何避免？
**答：** 常见模板：
```python
# 左闭右闭 [left, right]
left, right = 0, len(nums) - 1
while left <= right:
    mid = left + (right - left) // 2
    if nums[mid] == target: return mid
    elif nums[mid] < target: left = mid + 1
    else: right = mid - 1

# 左闭右开 [left, right)
left, right = 0, len(nums)
while left < right:
    mid = left + (right - left) // 2
    if nums[mid] == target: return mid
    elif nums[mid] < target: left = mid + 1
    else: right = mid
```
- 关键：明确区间定义，保持循环不变量

### 38. 二分查找的变体：查找第一个/最后一个等于目标的位置？
**答：**
```python
# 查找第一个等于target的位置
def find_first(nums, target):
    left, right = 0, len(nums) - 1
    ans = -1
    while left <= right:
        mid = left + (right - left) // 2
        if nums[mid] >= target:
            right = mid - 1
            if nums[mid] == target: ans = mid
        else:
            left = mid + 1
    return ans
```
- 核心：即使找到目标，也继续向左边/右边压缩区间

### 39. 什么是二分答案？适用什么场景？
**答：** 当问题的答案具有单调性时，可以用二分查找确定最优解。场景：
- 最小化最大值：如分配任务使最大工作量最小
- 最大化最小值：如选择位置使距离最小值最大
- 判定问题：给定一个值，能否满足条件？
- 步骤：确定答案范围 → 二分中间值 → 检查可行性 → 调整范围

### 40. 哈希查找和二分查找的对比？
**答：**
| 特性 | 哈希查找 | 二分查找 |
|------|----------|----------|
| 时间复杂度 | O(1)平均 | O(log n) |
| 空间复杂度 | O(n) | O(1) |
| 数据要求 | 无要求 | 必须有序 |
| 范围查询 | 不支持 | 支持 |
| 顺序统计 | 不支持 | 支持 |
| 适用场景 | 精确查找 | 有序数据、范围查询 |

### 41. 二叉搜索树的查找、插入、删除操作详细步骤？
**答：**
- **查找**：从根开始，小则左，大则右，直到找到或为空
- **插入**：找到合适的叶子位置插入
- **删除**：
  - 叶子节点：直接删除
  - 只有一个子节点：用子节点替代
  - 有两个子节点：找到右子树最小值（或左子树最大值）替代，然后删除那个节点

### 42. 平衡二叉搜索树（AVL/红黑树）为什么比普通BST重要？
**答：** 普通BST在有序数据插入时会退化为链表，O(n)时间复杂度。平衡树通过旋转操作保持平衡，保证O(log n)的最坏时间复杂度，确保性能稳定。

### 43. 什么是跳表查找？时间复杂度分析？
**答：** 跳表查找从最高层开始，向右比较，如果下一个节点值大于目标，则下降到下一层。每层跳过的节点数期望为2，因此期望时间复杂度O(log n)。空间换时间的典型应用。

### 44. 字符串查找中，KMP算法的核心思想是什么？
**答：** KMP（Knuth-Morris-Pratt）算法：
- **核心**：利用已匹配的信息，避免从头比较
- **next数组（前缀函数）**：记录模式串中每个位置的最长相等前后缀长度
- 匹配失败时，模式串向右滑动，使前缀对准已匹配的后缀
- 时间复杂度：O(m+n)，m为模式串长度，n为文本长度

### 45. Boyer-Moore字符串匹配算法与KMP的区别？
**答：**
- **KMP**：从左到右匹配，利用前缀信息，适合一般场景
- **Boyer-Moore**：从右到左匹配，利用坏字符规则和好后缀规则，跳过更多字符
- 实际应用中Boyer-Moore通常更快，但实现更复杂
- KMP更稳定，最坏O(m+n)；Boyer-Moore最坏O(mn)，但平均很好

## 四、递归与分治（46-55题）

### 46. 递归算法的三要素是什么？
**答：**
1. **终止条件**：递归何时停止，避免无限递归
2. **递归关系**：如何将大问题分解为小问题
3. **返回值**：递归函数返回什么，如何合并子问题结果

### 47. 递归和迭代的区别？如何相互转换？
**答：**
- **递归**：函数自我调用，代码简洁，但有栈溢出风险，有函数调用开销
- **迭代**：循环实现，无栈溢出风险，通常更高效
- **转换**：递归→迭代可用栈模拟调用栈；尾递归可优化为迭代

### 48. 什么是尾递归？为什么可以优化？
**答：** 尾递归是递归调用在函数最后一步执行，且返回值不依赖递归调用结果。编译器可以复用当前栈帧，避免栈增长，转化为迭代。但Python等语言不支持尾递归优化。

### 49. 分治算法（Divide and Conquer）的基本步骤？
**答：**
1. **分解**：将原问题分解为若干规模较小的子问题
2. **解决**：递归解决子问题（如果子问题足够小，直接求解）
3. **合并**：将子问题的解合并为原问题的解
- 经典应用：归并排序、快速排序、二分查找、Strassen矩阵乘法、最近点对

### 50. 主定理（Master Theorem）是什么？如何使用？
**答：** 用于求解分治递归式T(n) = aT(n/b) + f(n)的时间复杂度：
- 若f(n) = O(n^c)且c < log_b(a)，则T(n) = Θ(n^(log_b(a)))
- 若f(n) = Θ(n^(log_b(a)) * log^k n)，则T(n) = Θ(n^(log_b(a)) * log^(k+1) n)
- 若f(n) = Ω(n^c)且c > log_b(a)，则T(n) = Θ(f(n))
- 例：归并排序T(n)=2T(n/2)+O(n)，a=2,b=2,log_b(a)=1，f(n)=Θ(n^1)，得T(n)=Θ(n log n)

### 51. 归并排序的合并过程详细步骤？
**答：**
1. 创建临时数组，大小为两个子数组之和
2. 用两个指针分别遍历两个子数组
3. 每次取较小元素放入临时数组，对应指针后移
4. 当一个子数组遍历完后，将另一个子数组剩余元素全部放入
5. 将临时数组复制回原数组
- 时间复杂度：O(n)，空间复杂度：O(n)

### 52. 快速排序的分区过程详细步骤（Lomuto）？
**答：**
```python
def partition(arr, low, high):
    pivot = arr[high]  # 选最后一个为基准
    i = low - 1        # i指向小于基准的最后一个位置
    for j in range(low, high):
        if arr[j] <= pivot:
            i += 1
            arr[i], arr[j] = arr[j], arr[i]
    arr[i+1], arr[high] = arr[high], arr[i+1]
    return i + 1       # 基准最终位置
```

### 53. 什么是递归树？如何用它分析递归算法复杂度？
**答：** 递归树是将递归调用过程可视化为树结构：
- 每个节点代表一个子问题的代价
- 树高代表递归深度
- 每层代价之和代表该层总工作量
- 总代价 = 各层代价之和
- 例：归并排序递归树，每层代价都是O(n)，共log n层，总O(n log n)

### 54. 分治算法中"合并"步骤的复杂度如何影响总体复杂度？
**答：** 分治算法的复杂度取决于合并步骤：
- 如果合并是O(n)（如归并排序），总复杂度O(n log n)
- 如果合并是O(n²)（如某些矩阵问题），总复杂度可能O(n²)
- 优化合并步骤是提升分治算法效率的关键
- 有时可以用更聪明的方法避免显式合并（如快速排序的原地分区）

### 55. 大整数乘法（Karatsuba算法）如何用分治优化？
**答：** 传统O(n²)，Karatsuba利用分治：
- 将n位数分成两半：x = a*10^(n/2) + b, y = c*10^(n/2) + d
- xy = ac*10^n + (ad+bc)*10^(n/2) + bd
- 关键：ad+bc = (a+b)(c+d) - ac - bd，只需3次乘法（ac, bd, (a+b)(c+d)）
- 复杂度：T(n) = 3T(n/2) + O(n) = O(n^log₂3) ≈ O(n^1.585)

## 五、动态规划（56-70题）

### 56. 动态规划（DP）的核心思想是什么？与递归/分治的区别？
**答：** DP核心：**最优子结构** + **重叠子问题** + **无后效性**
- 将大问题分解为子问题，保存子问题解避免重复计算
- 与递归区别：DP是自底向上填表，递归是自顶向下
- 与分治区别：分治子问题不重叠，DP子问题重叠
- 与贪心区别：贪心局部最优即全局最优，DP需要比较所有子问题解

### 57. 动态规划解题的一般步骤？
**答：**
1. **定义状态**：dp[i]或dp[i][j]代表什么含义
2. **状态转移方程**：dp[i]如何从dp[i-1]等推导
3. **初始化**：边界条件
4. **遍历顺序**：确保计算dp[i]时所需状态已计算
5. **返回结果**：dp[n]或dp[n][m]等

### 58. 0-1背包问题的状态和转移方程？
**答：** dp[i][j] = 前i个物品，容量为j时的最大价值
```
dp[i][j] = max(dp[i-1][j], dp[i-1][j-weight[i]] + value[i])  if j >= weight[i]
         = dp[i-1][j]                                       otherwise
```
- 空间优化：滚动数组，一维dp[j]从大到小遍历

### 59. 完全背包和0-1背包的区别？状态转移有什么不同？
**答：** 完全背包每个物品可以选无限次：
```
# 0-1背包（一维，从大到小）
for j in range(capacity, weight[i]-1, -1):
    dp[j] = max(dp[j], dp[j-weight[i]] + value[i])

# 完全背包（一维，从小到大）
for j in range(weight[i], capacity+1):
    dp[j] = max(dp[j], dp[j-weight[i]] + value[i])
```
- 区别：遍历顺序相反，因为完全背包可以重复选

### 60. 最长公共子序列（LCS）的状态转移方程？
**答：** dp[i][j] = s1[0:i]和s2[0:j]的LCS长度
```
if s1[i-1] == s2[j-1]: dp[i][j] = dp[i-1][j-1] + 1
else: dp[i][j] = max(dp[i-1][j], dp[i][j-1])
```
- 时间复杂度：O(m*n)，空间复杂度可优化为O(min(m,n))

### 61. 最长递增子序列（LIS）的O(n log n)解法？
**答：** 维护一个数组tails，tails[i]表示长度为i+1的递增子序列的最小末尾元素：
```python
def lengthOfLIS(nums):
    tails = []
    for num in nums:
        idx = bisect_left(tails, num)
        if idx == len(tails): tails.append(num)
        else: tails[idx] = num
    return len(tails)
```
- 用二分查找确定num的位置，时间复杂度O(n log n)

### 62. 编辑距离（Edit Distance / Levenshtein Distance）的状态转移？
**答：** dp[i][j] = word1[0:i]转换为word2[0:j]的最少操作数
```
if word1[i-1] == word2[j-1]: dp[i][j] = dp[i-1][j-1]
else: dp[i][j] = 1 + min(dp[i-1][j],    # 删除
                         dp[i][j-1],    # 插入
                         dp[i-1][j-1])  # 替换
```

### 63. 矩阵链乘法（Matrix Chain Multiplication）如何用DP优化？
**答：** dp[i][j] = 计算矩阵Ai到Aj的最小乘法次数
```
dp[i][j] = min(dp[i][k] + dp[k+1][j] + p[i-1]*p[k]*p[j]) for k in [i, j-1]
```
- 区间DP，按区间长度从小到大填表
- 时间复杂度：O(n³)，空间复杂度：O(n²)

### 64. 什么是状态压缩DP？适用什么场景？
**答：** 用二进制位表示集合状态，将多维状态压缩为整数。适用场景：
- 子集问题：如旅行商问题（TSP）、集合覆盖
- 棋盘覆盖问题
- 状态数2^n，n通常≤20
- 例：TSP中dp[mask][i]表示已访问mask集合，当前在i城市的最小距离

### 65. 树形DP是什么？如何解决树上问题？
**答：** 在树结构上进行动态规划，通常用DFS后序遍历：
- 先递归处理所有子节点，得到子树的DP值
- 再根据子节点结果计算当前节点的DP值
- 应用：树的最大独立集、树的直径、换根DP、树背包

### 66. 什么是数位DP？解决什么问题？
**答：** 用于统计满足特定条件的数字个数，通常范围很大（如1到10^18）。方法：
- 将数字按位分解，从高位到低位DP
- 状态通常包括：当前位、是否受限（tight）、是否有前导零、其他条件
- 用记忆化搜索实现，避免重复计算

### 67. 股票买卖问题的DP解法（最多K次交易）？
**答：** dp[i][k][0/1] = 第i天，最多k次交易，持有/不持有股票的最大利润
```
dp[i][k][0] = max(dp[i-1][k][0], dp[i-1][k][1] + prices[i])  # 卖出
dp[i][k][1] = max(dp[i-1][k][1], dp[i-1][k-1][0] - prices[i])  # 买入
```
- k=1和k=∞可简化为一维DP

### 68. 什么是区间DP？有什么经典问题？
**答：** 区间DP是状态表示区间的DP，通常按区间长度从小到大计算：
- 经典问题：矩阵链乘、石子合并、括号匹配、回文串划分、最优二叉搜索树
- 状态：dp[i][j] = 区间[i,j]的最优解
- 转移：枚举分割点k，dp[i][j] = min/max(dp[i][k] + dp[k+1][j] + cost)

### 69. 什么是DAG上的DP？与树形DP的区别？
**答：** 在DAG（有向无环图）上进行DP，按拓扑排序顺序计算：
- 树形DP是DAG的特例（树是DAG）
- DAG可能有多个父节点，需要合并多个来源的状态
- 应用：最长路径、最短路径、方案计数
- 必须先拓扑排序，确保计算节点时所有前驱已计算

### 70. DP的空间优化技巧有哪些？
**答：**
- **滚动数组**：只保留最近两行/几行，如0-1背包一维优化
- **状态压缩**：用位运算表示集合
- **逆序遍历**：避免覆盖未使用的状态（如0-1背包）
- **原地修改**：如果新状态只依赖旧状态，可复用数组
- **对称性优化**：利用问题对称性减少状态数

## 六、图论算法（71-82题）

### 71. 图的深度优先搜索（DFS）和广度优先搜索（BFS）的区别？
**答：**
| 特性 | DFS | BFS |
|------|-----|-----|
| 数据结构 | 栈（递归/显式） | 队列 |
| 搜索顺序 | 先深入到底，再回溯 | 逐层扩展 |
| 空间复杂度 | O(h)，h为深度 | O(w)，w为最大宽度 |
| 适用场景 | 连通性、拓扑排序、回溯 | 最短路径（无权图）、层级遍历 |
| 是否找到最短路径 | 否 | 是（无权图） |

### 72. Dijkstra算法的原理和时间复杂度？适用条件？
**答：** 单源最短路径，贪心策略：
1. 初始化距离数组，起点为0，其余为∞
2. 每次选距离最小的未确定节点，标记为确定
3. 用该节点松弛所有邻接边
4. 重复直到所有节点确定
- **时间复杂度**：O((V+E)log V)（优先队列实现）
- **适用条件**：边权非负
- **正确性证明**：每次确定的节点距离已是最短（反证法）

### 73. 为什么Dijkstra算法不能处理负权边？
**答：** Dijkstra的贪心策略假设：一旦确定最短距离，不会再被更新。但负权边可能导致：
- 已确定节点的最短路径可能通过负权边变得更短
- 破坏贪心选择的最优子结构性质
- 反例：A→B(5), A→C(3), C→B(-2)，Dijkstra先确定B=5，但实际最短是A→C→B=1

### 74. Bellman-Ford算法和SPFA算法是什么？
**答：**
- **Bellman-Ford**：松弛所有边V-1次，可检测负环，时间复杂度O(VE)
- **SPFA**：Bellman-Ford的队列优化，只松弛距离更新的节点的邻接边，平均O(E)，最坏O(VE)
- 适用：有负权边、需要检测负环的场景

### 75. Floyd-Warshall算法解决什么问题？时间复杂度？
**答：** 多源最短路径（所有点对最短路径）：
- 状态：dp[k][i][j] = 经过节点{1..k}，i到j的最短距离
- 转移：dp[k][i][j] = min(dp[k-1][i][j], dp[k-1][i][k] + dp[k-1][k][j])
- 空间优化：dp[i][j] = min(dp[i][j], dp[i][k] + dp[k][j])，k作为最外层循环
- 时间复杂度：O(V³)，空间复杂度：O(V²)
- 适用：稠密图、需要所有点对最短路径

### 76. 最小生成树（MST）的Kruskal和Prim算法对比？
**答：**
| 特性 | Kruskal | Prim |
|------|---------|------|
| 策略 | 按边权排序，选不形成环的边 | 从起点扩展，选最小边连接树外节点 |
| 数据结构 | 并查集 | 优先队列（最小堆） |
| 时间复杂度 | O(E log E) | O((V+E)log V) |
| 适用场景 | 稀疏图 | 稠密图 |
| 空间复杂度 | O(E) | O(V) |

### 77. 什么是最短路径树（SPT）？与最小生成树的区别？
**答：**
- **SPT**：从源点到所有其他节点的最短路径组成的树（Dijkstra/BFS结果）
- **MST**：连接所有节点且总边权最小的树
- 区别：SPT保证源点到各点距离最短，MST保证总边权最小；SPT依赖源点，MST不依赖
- 例：三角形边权1,2,3，MST选1+2=3，SPT从某点出发可能选1+3=4

### 78. 拓扑排序的Kahn算法和DFS算法？
**答：**
- **Kahn算法（BFS）**：计算所有节点入度，入度为0的入队，出队时减少邻接节点入度，重复直到队列为空。如果还有节点未处理，说明有环。
- **DFS算法**：DFS后序遍历，逆序输出。如果DFS遇到已访问节点，说明有环。

### 79. 强连通分量（SCC）是什么？如何求？
**答：** 强连通分量是有向图中任意两点互相可达的最大子图。求法：
- **Kosaraju算法**：两次DFS，第一次记录完成顺序，第二次在转置图上按逆序DFS
- **Tarjan算法**：一次DFS，用dfn（访问序号）和low（能到达的最小dfn）判断，low==dfn时弹出栈中节点为一个SCC
- **时间复杂度**：O(V+E)

### 80. 二分图（Bipartite Graph）是什么？如何判断？
**答：** 顶点可分为两组，组内无边，边只存在于两组之间。判断方法：
- **染色法**：BFS/DFS，尝试用两种颜色染色，相邻节点颜色不同。如果冲突则不是二分图。
- 等价条件：不含奇数长度环
- 应用：匹配问题、任务分配

### 81. 匈牙利算法（Hungarian Algorithm）解决什么问题？
**答：** 解决二分图最大匹配问题：
- 从左部未匹配节点出发，找增广路径（交替的未匹配边-匹配边-未匹配边...）
- 找到增广路径后，翻转路径上的匹配状态，匹配数+1
- 时间复杂度：O(VE)
- 应用：任务分配、婚姻匹配、资源分配

### 82. 网络流中的最大流问题如何解决？Ford-Fulkerson方法？
**答：** Ford-Fulkerson方法：
1. 在残余网络中找增广路径（从源到汇有剩余容量的路径）
2. 沿增广路径增加流量，更新残余网络
3. 重复直到无增广路径
- **Edmonds-Karp**：用BFS找最短增广路径，O(VE²)
- **Dinic**：分层图+阻塞流，O(V²E)
- **最小割最大流定理**：最大流等于最小割的容量

## 七、字符串算法（83-90题）

### 83. 字符串哈希（Rabin-Karp）的原理？
**答：** 将字符串看作某个进制（如26或131）的数字，取模避免溢出：
- hash(s) = (s[0]*p^(n-1) + s[1]*p^(n-2) + ... + s[n-1]) mod M
- 滑动窗口时：hash_new = (hash_old - s[0]*p^(n-1)) * p + s[n] mod M
- 双哈希（两个不同模数）减少冲突
- 应用：字符串匹配、判断子串是否相等

### 84. KMP算法中next数组（前缀函数）如何计算？
**答：**
```python
def compute_prefix(pattern):
    n = len(pattern)
    pi = [0] * n
    for i in range(1, n):
        j = pi[i-1]
        while j > 0 and pattern[i] != pattern[j]:
            j = pi[j-1]
        if pattern[i] == pattern[j]:
            j += 1
        pi[i] = j
    return pi
```
- pi[i] = pattern[0:i+1]的最长相等真前缀和真后缀的长度
- 计算过程也是KMP匹配过程，时间复杂度O(m)

### 85. Trie树（前缀树）的构建和查询过程？
**答：**
- **构建**：从根开始，对每个字符串的每个字符，如果不存在对应子节点则创建，最后标记结束节点
- **查询**：从根开始按字符遍历，如果某字符不存在则失败，到达末尾检查是否结束节点
- **时间复杂度**：插入/查询O(L)，L为字符串长度
- **空间复杂度**：O(总字符数 * 字符集大小)

### 86. AC自动机（Aho-Corasick）是什么？与KMP的关系？
**答：** AC自动机是多模式串匹配算法（一次文本匹配多个模式）：
- 构建：将所有模式串构建Trie树，添加失败指针（类似KMP的next数组）
- 匹配：在Trie上按文本字符遍历，失败时沿失败指针跳转
- 时间复杂度：O(n + m + z)，n为文本长度，m为模式总长度，z为匹配次数
- 与KMP关系：KMP是单模式串的AC自动机特例

### 87. 后缀数组（Suffix Array）和后缀树（Suffix Tree）是什么？
**答：**
- **后缀数组**：字符串所有后缀按字典序排序后的起始位置数组
- **后缀树**：所有后缀的压缩Trie，边表示子串而非单个字符
- 应用：最长重复子串、最长公共子串、模式匹配
- 构建：DC3算法O(n)，或倍增算法O(n log n)

### 88. Manacher算法解决什么问题？时间复杂度？
**答：** 求字符串的最长回文子串，线性时间O(n)：
- 预处理：插入分隔符（如#），将奇偶回文统一为奇数长度
- 维护中心c和最右边界r，利用对称性减少计算
- 如果i在r内，p[i] = min(p[2*c-i], r-i)
- 然后尝试扩展，更新c和r

### 89. Z函数（Z-Algorithm）是什么？
**答：** Z[i] = s[i:]与s的最长公共前缀长度。计算：
- 维护区间[l,r]使得s[l:r]是s的前缀
- 如果i在r内，Z[i] = min(Z[i-l], r-i+1)
- 然后尝试扩展，更新l和r
- 时间复杂度：O(n)
- 应用：模式匹配（拼接pattern#$text）、字符串压缩

### 90. 滚动哈希（Rolling Hash）的冲突概率如何控制？
**答：**
- 使用大质数作为模数（如10^9+7, 10^9+9）
- 使用双哈希（两个不同模数），冲突概率降至1/M1*1/M2
- 使用随机基数，避免被恶意数据攻击（Hash Collision攻击）
- 对于安全性要求高的场景，使用加密哈希（如SHA-256），但速度较慢

## 八、贪心算法与回溯（91-100题）

### 91. 贪心算法（Greedy Algorithm）的核心思想？与DP的区别？
**答：** 贪心：每一步做出局部最优选择，希望达到全局最优。与DP区别：
- **贪心**：不回溯，一旦选择不可撤销；DP保存所有子问题解，可回溯比较
- **贪心**：需要证明局部最优能导致全局最优；DP不需要证明，只要满足最优子结构
- **贪心更快**：通常O(n log n)或O(n)；DP通常O(n²)或更高
- **适用场景**：活动选择、哈夫曼编码、最小生成树、单源最短路径（Dijkstra）

### 92. 活动选择问题（Activity Selection）的贪心策略？
**答：** 选最多不重叠活动：
- 按结束时间排序
- 每次选择结束时间最早且不与已选活动冲突的活动
- 证明：设贪心选择第一个活动，任何最优解都可以调整为包含这个活动
- 时间复杂度：O(n log n)（排序）

### 93. 哈夫曼编码（Huffman Coding）的构建过程？
**答：**
1. 统计字符频率，每个字符作为叶子节点
2. 每次选频率最小的两个节点，合并为新节点（频率为两者之和）
3. 新节点加入优先队列，重复直到只剩一个节点（根）
4. 左分支标0，右分支标1，从根到叶子的路径即为编码
- 时间复杂度：O(n log n)，带权路径长度最小（最优前缀编码）

### 94. 什么是回溯算法（Backtracking）？与DFS的关系？
**答：** 回溯是DFS的特例，用于搜索所有（或满足条件的）解：
- 在DFS过程中，如果发现当前路径不可能得到解，则剪枝返回
- 与DFS区别：DFS通常遍历所有节点；回溯有剪枝，只探索可能的分支
- 框架：选择 → 递归 → 撤销选择（恢复状态）
- 应用：N皇后、数独、全排列、子集、组合、图着色

### 95. N皇后问题的回溯解法？如何优化？
**答：**
- 逐行放置皇后，每行尝试每列
- 检查冲突：同列、对角线（row-col相同或row+col相同）
- **优化**：
  - 位运算优化（用一个整数表示列和对角线占用状态）
  - 对称性剪枝（第一行只放一半列）
  - 预处理：每行可用列用位掩码表示
- 时间复杂度：O(n!)，但剪枝后实际很快

### 96. 全排列（Permutation）的回溯算法？
**答：**
```python
def permute(nums):
    result = []
    def backtrack(path, used):
        if len(path) == len(nums):
            result.append(path[:])
            return
        for i in range(len(nums)):
            if used[i]: continue
            used[i] = True
            path.append(nums[i])
            backtrack(path, used)
            path.pop()
            used[i] = False
    backtrack([], [False]*len(nums))
    return result
```
- 去重：先排序，跳过相同且前一个未使用的元素

### 97. 子集（Subset）问题的回溯和位运算解法？
**答：**
- **回溯**：每个元素选或不选，递归生成所有子集
- **位运算**：用n位二进制数表示选或不选，第i位为1表示选nums[i]
  ```python
  def subsets(nums):
      n = len(nums)
      result = []
      for mask in range(1 << n):
          subset = [nums[i] for i in range(n) if mask & (1 << i)]
          result.append(subset)
      return result
  ```
- 时间复杂度：O(n * 2^n)

### 98. 分支限界法（Branch and Bound）与回溯的区别？
**答：**
- **回溯**：深度优先，找到所有解，剪枝基于可行性（是否满足约束）
- **分支限界**：广度优先或最佳优先，找最优解，剪枝基于界限（当前最优解不可能比已知最优更好）
- 界限函数：计算当前部分解的下界（最小化问题）或上界（最大化问题）
- 应用：旅行商问题、0-1背包（用DP或贪心计算界限）

### 99. 剪枝技巧在回溯/DFS中如何应用？
**答：**
- **可行性剪枝**：当前选择不满足约束，直接返回
- **最优性剪枝**：当前部分解已不可能优于已知最优解，返回
- **对称性剪枝**：利用对称性减少搜索空间
- **记忆化**：保存已计算的状态，避免重复搜索（DFS+DP）
- **排序预处理**：先处理约束强的变量，减少分支
- **位运算优化**：用整数状态代替数组，加速判断

### 100. 如何分析递归/回溯算法的时间复杂度？
**答：**
- **递归树法**：画递归树，计算节点数和每层工作量
- **主定理**：适用于分治型递归
- **状态数 × 转移代价**：如DP状态数为O(n²)，转移O(1)，总O(n²)
- **最坏情况**：回溯通常是指数级，但剪枝后可能很好
- **摊还分析**：某些操作看似昂贵，但均摊后便宜
- **实际测试**：小规模数据验证理论分析
